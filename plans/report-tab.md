# Feature: Report Tab — Equity Research Report

## Overview

Add a "Report" tab that generates a professional, sell-side-style equity research report combining all analysis into a single document. The report has four sections: (1) Key Statistics header, (2) AI-generated investment narrative, (3) financial model table with actuals + projections, and (4) peer comparison matrix. Available as on-screen HTML preview and downloadable PDF via weasyprint.

## Architecture Decision: Narrative Generation Timing

The PRD states "All data sourced from the same API response — no additional calls." The report narrative is generated by Claude during the main analysis pipeline (Phase 2 of the `/analyze-stream/` route), running concurrently with the existing bull/bear and industry Claude calls. This adds zero extra wall-clock time since Claude calls already dominate the timeline and all three run in parallel via `asyncio.gather`.

## Files to Create

- `backend/report_generator.py` — Claude AI generates the 2-page sell-side narrative
- `frontend/js/report.js` — Renders the report preview (all 4 sections)

## Files to Modify

- `main.py` — Add ReportNarrative Pydantic model, add `generate_report_narrative()` to Phase 2 gather, add `report_narrative` to AnalysisResponse, add `POST /report/pdf/{ticker}` endpoint
- `frontend/index.html` — Add Report tab button + tab container HTML, add `<script>` tag for report.js
- `frontend/js/app.js` — Add 'report' to TAB_IDS array, call `renderReport(data)` in form submit handler
- `frontend/style.css` — Report-specific styles (key stats box, narrative prose, print-friendly tables, PDF button)

## Data Flow

```
[Existing analysis data] ──┐
                            ├── Claude (concurrent) ──> report_narrative string
[10-K + transcript text] ──┘

main.py Phase 2:
  asyncio.gather(
    analyze(),              # existing
    analyze_industry(),     # existing
    generate_report_narrative()  # NEW — runs in parallel
  )

AnalysisResponse includes:
  report_narrative: Optional[str]   # AI-generated narrative (HTML-safe markdown)

Frontend:
  renderReport(data) builds the on-screen preview from:
    - data.report_narrative          (AI text)
    - data.dcf_model.actuals         (historical financials)
    - data.dcf_model.*               (projection assumptions)
    - data.comps.peers               (peer matrix)
    - data.current_price, data.intrinsic_value, etc. (key stats)
    - data.finviz.*                  (analyst target, metrics)

PDF export:
  POST /report/pdf/{ticker}  body={...all data...}
  Server builds self-contained HTML → weasyprint → PDF bytes → download
```

### Prompt Construction

The prompt includes:
- Ticker and sector
- 10-K excerpts (MD&A + Risk Factors) — reuse same snippets
- Transcript excerpts
- Bull/bear analysis results (from Phase 2 — pass as context)
- Key financial metrics (revenue, margins, EPS, FCF from DCF actuals)
- DCF intrinsic value and upside/downside
- Comps median multiples and implied prices
- DDM values (if available)
- Finviz analyst target and recommendation

### Important: Sequential dependency

The report narrative needs the analysis results (bull/bear, rating) as input context. This means it CANNOT run fully in parallel with `analyze()`. Two options:

**Option A (chosen):** Run it after `analyze()` completes, as a third sequential step. The narrative call takes ~3-5 seconds. Total latency increase: ~3-5s.

**Option B:** Run it in parallel without analysis context, relying only on 10-K/transcript/financials. Faster but lower quality since it can't reference the bull/bear conclusions.

We choose **Option A** because report quality matters more than 3s of latency, and the report narrative enriches the overall product. The pipeline becomes:

```
Phase 0: sector classification
Phase 1: concurrent data fetch (10-K, transcript, DCF, DDM, financials, finviz, news)
Phase 2: concurrent Claude calls (analyze + analyze_industry)
Phase 3: report narrative (uses Phase 2 results as context)  ← NEW
Phase 4: assemble response
```

Actually, looking more carefully: we can avoid the sequential dependency by building the financial/valuation summary from Phase 1 data (DCF results, financials, comps) which are available before Phase 2. The narrative doesn't strictly need the bull/bear text — it can independently assess the company from the same source documents. Let's keep it concurrent:

```
Phase 2: concurrent Claude calls (analyze + analyze_industry + report_narrative)
```

The report prompt will include financial data context (DCF IV, comps medians, key financials) but not the bull/bear analysis output. Claude will independently generate its narrative from the 10-K, transcript, and quantitative data.

### 2c. Build financial context string (between Phase 1 and Phase 2)

After Phase 1 data fetch completes, build a pre-formatted string summarizing the quantitative data for the report prompt:

```python
def _build_report_context(dcf_data, ddm_data, comps_data, financials_data, finviz_data):
    """Build a text summary of financial data for the report narrative prompt."""
    lines = []
    if dcf_data:
        lines.append(f"Current Price: ${dcf_data.current_price}")
        lines.append(f"DCF Intrinsic Value: ${dcf_data.intrinsic_value} ({dcf_data.upside_downside:+.1f}%)")
        last = dcf_data.actuals[-1]
        lines.append(f"Latest FY Revenue: ${last.revenue/1e9:.2f}B")
        lines.append(f"Latest FY Net Income: ${last.net_income/1e9:.2f}B")
        lines.append(f"Latest FY EPS: ${last.eps:.2f}")
        lines.append(f"Latest FY FCF: ${last.fcf/1e9:.2f}B")
    # ... similar for DDM, comps, finviz ...
    return "\n".join(lines)
```

### 2e. Add PDF endpoint

```python
@app.post("/report/pdf/{ticker}", tags=["report"])
async def generate_pdf(ticker: str, request: Request) -> StreamingResponse:
    """
    Accept the full analysis data as JSON body, render a self-contained
    HTML document, convert to PDF via weasyprint, and return the file.
    """
    body = await request.json()
    html = _build_report_html(ticker, body)
    pdf_bytes = await asyncio.to_thread(_render_pdf, html)
    return StreamingResponse(
        io.BytesIO(pdf_bytes),
        media_type="application/pdf",
        headers={"Content-Disposition": f'attachment; filename="{ticker}_equity_report.pdf"'},
    )
```

The `_build_report_html()` function creates a complete HTML document with inline CSS (no external dependencies) containing all 4 report sections. The `_render_pdf()` function calls `weasyprint.HTML(string=html).write_pdf()`.

```javascript
function renderReport(data) {
```

```javascript
function renderNarrativeSections(narrative) {
  const container = document.getElementById('report-narrative');
  const sections = [
    { title: 'Company Overview', content: narrative.company_overview },
    { title: 'Recent Performance & Earnings', content: narrative.recent_performance },
    { title: 'Forward Outlook & Growth Drivers', content: narrative.forward_outlook },
    { title: 'Balance Sheet & Capital Allocation', content: narrative.balance_sheet_capital },
    { title: 'Valuation & Recommendation', content: narrative.valuation_recommendation },
  ];
  container.innerHTML = sections.map(s => `
    <div class="report-section">
      <h3 class="report-section-title">${escapeHtml(s.title)}</h3>
      <div class="report-prose">${parseMarkdown(s.content)}</div>
    </div>
  `).join('');
}
```

### Download PDF Button

```javascript
function wireDownloadButton(data) {
  document.getElementById('report-download-btn').onclick = async () => {
    const btn = document.getElementById('report-download-btn');
    btn.disabled = true;
    btn.textContent = 'Generating PDF...';
    try {
      const res = await fetch('/report/pdf/' + encodeURIComponent(data.ticker), {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(data),
      });
      if (!res.ok) throw new Error('PDF generation failed');
      const blob = await res.blob();
      const url = URL.createObjectURL(blob);
      const a = document.createElement('a');
      a.href = url;
      a.download = data.ticker + '_equity_report.pdf';
      a.click();
      URL.revokeObjectURL(url);
    } catch (err) {
      alert('PDF download failed: ' + err.message);
    } finally {
      btn.disabled = false;
      btn.textContent = 'Download PDF';
    }
  };
}
```

## Edge Cases

- **No FMP key:** Report narrative still generates from 10-K/transcript. Financial model table shows "unavailable." Peer matrix may still show Finviz data.
- **Claude API failure for report:** `report_available = False`, tab shows "unavailable" message. Other tabs unaffected.
- **Very long narrative:** Claude output is capped at 4096 tokens. Narrative sections are structured to fit within this budget.
- **weasyprint missing:** PDF button shows install instructions. On-screen preview still works perfectly.
